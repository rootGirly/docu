## Why this exists

I started documenting on Medium and GitHub Gists, but I needed a **self-contained, version-controlled** environment.

- **Control:** I own the data, the format, and the history. No platform dependencies.
- **Accessibility:** If I want to show a command snippet to my cousin, he don't need a login. It's just Markdown.
- **Methodology:** I'm building my own standard operating procedure (SOP) for homelab security, automation, and incident response.
- **Efficiency:** Instead of re-searching the same `iptables` rules or `docker-compose` configs, I write them here. **Past Me** did the hard work; **Present Me** just executes the notes.

##  How to use it

1. **Search:** Use `⌘K` (or your editor's search) to find specific commands or configs.
2. **Verify:** Always test commands in a safe environment (VM/Container) before applying to production.
3. **Learn:** Every entry includes the **Why** (context/threat model) and the **How** (implementation).
